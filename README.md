# 🎓 CCU Invite — приглашения и контроль входа

Мобильная веб-платформа для **Колледжа Каспийского университета** для церемонии
вручения дипломов. Позволяет заранее внести студентов и их родителей/родственников,
сгенерировать именные **пригласительные билеты с QR-кодом** и на входе быстро
отмечать присутствие сканированием камерой телефона.

> Места ограничены — в зал попадают только заранее заявленные гости.

## Возможности

- 👨‍🎓 Добавление **студентов** и привязка **родителей/родственников** (с лимитом на студента)
- 🎟 Генерация **именного пригласительного** — красивая ссылка в стиле CCU с QR; действует **1 месяц**
- 📷 **Сканер на входе** прямо в браузере телефона: навёл камеру → увидел гостя → «Подтвердить присутствие»
- ✅ Автоматическая отметка в базе, защита от повторов, фильтры и поиск
- 📊 Сводка по мероприятию: сколько приглашено / пришло, заполняемость зала
- 👥 Роли: **администратор** (управляет данными) и **контролёр/сканер** (только вход)

## Технологии

Flask 3 · SQLite · Flask-Login · Flask-WTF (CSRF) · `qrcode` · `html5-qrcode` ·
mobile-first CSS без сборки.

## Быстрый старт

```bash
# 1. зависимости
python3 -m venv venv
source venv/bin/activate          # Windows: venv\Scripts\activate
pip install -r requirements.txt

# 2. база + демо-данные и учётки
python3 -m scripts.seed --demo

# 3. запуск
python3 run.py                    # → http://localhost:5000
```

**Демо-вход:**
- Администратор — `admin` / `admin123`
- Контролёр (сканер) — `scanner` / `scan123`

> ⚠️ Обязательно смените пароли и `SECRET_KEY` перед реальным использованием
> (скопируйте `.env.example` в `.env`).

## 📷 Камера на телефоне — важно

Браузеры дают доступ к камере только по **HTTPS** или на `localhost`. Чтобы
сканер заработал на телефоне в локальной сети, запустите с самоподписанным
сертификатом:

```bash
SSL_ADHOC=1 python3 run.py        # затем откройте https://<ip-компьютера>:5000
```

Альтернатива — туннель (ngrok / cloudflared), который сразу даёт https-адрес.
Для боевого домена (например `invite.ccu.kz`) укажите его в `.env`
(`PUBLIC_BASE_URL`) — тогда ссылки и QR будут вести на правильный адрес.

## Как пользоваться

1. **Администратор** → «Студенты» → добавляет студента.
2. В карточке студента добавляет родителей/родственников.
3. Жмёт **«Сгенерировать пригласительное»** → копирует ссылку и отправляет
   родителю (WhatsApp/SMS/почта). Ссылка действует месяц.
4. Родитель открывает ссылку → видит **билет с QR**.
5. На входе **контролёр** заходит в «Сканер», включает камеру, наводит на QR,
   проверяет данные и жмёт **«Подтвердить присутствие»**.

## Конфигурация (`.env`)

| Переменная | По умолчанию | Назначение |
|-----------|--------------|-----------|
| `SECRET_KEY` | — | секрет Flask (**смените!**) |
| `PUBLIC_BASE_URL` | — | домен для ссылок/QR (напр. `https://invite.ccu.kz`) |
| `INVITE_VALID_DAYS` | `30` | срок действия приглашения |
| `MAX_GUESTS_PER_STUDENT` | `3` | лимит гостей на студента |
| `VENUE_CAPACITY` | `0` | вместимость зала (0 = без лимита) |
| `DISPLAY_TZ_OFFSET` | `5` | часовой пояс отображения (UTC+5) |
| `SSL_ADHOC` | — | `1` → запуск по HTTPS локально |

## 🚀 Деплой (Railway)

Проект готов к размещению на [Railway](https://railway.app) «из коробки»
(`railway.json`, `gunicorn`, release-скрипт, HTTPS-прокси, SQLite/Postgres).
На Railway **камера сканера работает на телефоне сразу** — сайт отдаётся по HTTPS.

Полная пошаговая инструкция: **[DEPLOY.md](DEPLOY.md)**.

Коротко: залить на GitHub → New Project from repo → подключить Volume `/data`
(`DATABASE_URL=sqlite:////data/ccu.db`) или Postgres → задать `SECRET_KEY`,
`FLASK_ENV=production`, `ADMIN_PASSWORD` → Generate Domain. Готово.
