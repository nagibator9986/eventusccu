# 🚀 Деплой на Railway

Пошаговая инструкция, как разместить CCU Invite на [Railway](https://railway.app).
Проект уже полностью подготовлен: `railway.json`, `Procfile`, `gunicorn`,
release-скрипт, поддержка прокси (HTTPS) и SQLite/Postgres.

> ✅ **Бонус Railway:** сайт по умолчанию отдаётся по **HTTPS**, поэтому **камера
> сканера сразу работает на телефоне** — никаких самоподписанных сертификатов.

---

## ⚠️ Самое важное про базу данных

Файловая система контейнера на Railway **эфемерная** — при каждом редеплое она
обнуляется. Если хранить SQLite в обычной папке, **все студенты, гости и отметки
пропадут**. Поэтому есть два варианта (выберите один):

- **Вариант A (просто): SQLite на Volume.** Подключаете том и кладёте базу туда.
- **Вариант B (надёжнее): PostgreSQL.** Плагин Railway, база живёт отдельно.

Оба варианта поддержаны кодом из коробки.

## Шаг 1. Залить код на GitHub

```bash
cd "ccu/zhigerhuinyaqre"
git init
git add .
git commit -m "CCU Invite — готов к деплою"
git branch -M main
git remote add origin https://github.com/<ваш-логин>/ccu-invite.git
git push -u origin main
```

`.gitignore` уже исключает `instance/`, `*.db` и `.env` — секреты и локальная база
в репозиторий не попадут.

---

## Шаг 2. Создать проект на Railway

1. [railway.app](https://railway.app) → **New Project** → **Deploy from GitHub repo**.
2. Выберите репозиторий. Railway сам определит Python (Nixpacks) и соберёт проект
   по `railway.json` (сервер `gunicorn`, healthcheck `/healthz`).

---

## Шаг 3. База данных

### Вариант A — SQLite на Volume (рекомендуется для простоты)

1. В сервисе откройте вкладку **Variables → New Volume** (или **Settings → Volumes**).
2. Создайте Volume с **Mount path** = `/data`.
3. В **Variables** добавьте переменную:
   ```
   DATABASE_URL = sqlite:////data/ccu.db
   ```
   (Именно **четыре** слэша — это абсолютный путь `/data/ccu.db`.)

### Вариант B — PostgreSQL

1. В проекте → **New → Database → Add PostgreSQL**.
2. Railway сам пробросит переменную `DATABASE_URL` в ваш сервис — больше ничего
   делать не нужно (код сам приведёт `postgres://` к `postgresql://`).

---

## Шаг 4. Переменные окружения (Variables)

Добавьте в сервисе (**Variables**):

| Переменная | Значение | Обязательно |
|-----------|----------|-------------|
| `SECRET_KEY` | длинная случайная строка (см. ниже) | **да** |
| `FLASK_ENV` | `production` | **да** |
| `ADMIN_PASSWORD` | пароль администратора | желательно |
| `SCANNER_PASSWORD` | пароль контролёра-сканера | желательно |
| `DATABASE_URL` | `sqlite:////data/ccu.db` (для варианта A) | для варианта A |
| `MAX_GUESTS_PER_STUDENT` | напр. `3` | нет |
| `VENUE_CAPACITY` | вместимость зала, напр. `200` | нет |
| `INVITE_VALID_DAYS` | `30` | нет |
| `DISPLAY_TZ_OFFSET` | `5` (Казахстан) | нет |

Сгенерировать `SECRET_KEY` локально:
```bash
python3 -c "import secrets; print(secrets.token_urlsafe(48))"
```

> Если не задать `ADMIN_PASSWORD`, на **первом** деплое пароль администратора
> **сгенерируется автоматически** и будет напечатан в логах деплоя (Deploy Logs).
> Откройте логи, найдите блок «СГЕНЕРИРОВАН пароль администратора», войдите и
> смените пароль.

---

## Шаг 5. Домен и ссылки-приглашения

1. **Settings → Networking → Generate Domain** — получите адрес вида
   `ccu-invite.up.railway.app`.
2. Ссылки-приглашения и QR заработают сразу: приложение определяет HTTPS-домен
   автоматически (через `ProxyFix`).
3. *(необязательно, но аккуратнее)* добавьте переменную, чтобы зафиксировать домен
   в ссылках:
   ```
   PUBLIC_BASE_URL = https://ccu-invite.up.railway.app
   ```
   Это особенно полезно, если позже подключите собственный домен.

После изменения переменных Railway передеплоит сервис автоматически.

## 📥 Автоматическая загрузка списка выпускников

Список из `data/graduates.xlsx` загружается в базу **автоматически при каждом
деплое** (скрипт `scripts/release.py`). Импорт **идемпотентный** — повторные
деплои не создают дубликатов. Для каждого гостя сразу генерируется
пригласительное (готовая ссылка с QR).

- Обновить список: замените `data/graduates.xlsx`, закоммитьте, запушьте —
  Railway передеплоит и догрузит новых.
- Отключить автозагрузку: переменная `IMPORT_ON_DEPLOY=0`.
- Не генерировать ссылки автоматически: `IMPORT_GENERATE_INVITES=0`.
- Загрузить вручную локально: `python -m scripts.import_data` (или `--dry-run`).

> 🔒 **Приватность:** файл содержит реальные ФИО студентов и родителей. Если
> репозиторий **публичный** — эти данные станут общедоступны. Рекомендуется
> сделать репозиторий **приватным** (GitHub → Settings → Danger Zone → Change
> visibility → Private) перед пушем с данными.

## Шаг 6. Проверка

1. Откройте `https://<ваш-домен>/healthz` → должно быть `{"status":"ok"}`.
2. Зайдите как администратор, добавьте студента и гостя, сгенерируйте
   пригласительное, откройте билет — QR должен указывать на ваш домен.
3. На телефоне откройте `https://<ваш-домен>/scan`, войдите как контролёр,
   включите камеру и отсканируйте билет — **камера работает, т.к. HTTPS**.

## Что уже сделано в коде для прода

- **`gunicorn`** как WSGI-сервер (`railway.json` / `Procfile`), 1 worker + 8 threads —
  оптимально для SQLite (без межпроцессных блокировок записи).
- **`scripts/release.py`** запускается перед стартом: создаёт таблицы и
  администратора/сканера из переменных (идемпотентно).
- **`ProxyFix`** — корректный `https` и хост за прокси Railway (secure-cookie и
  ссылки-приглашения).
- **SQLite в режиме WAL** + `busy_timeout` + `foreign_keys=ON` — устойчивость к
  параллельным запросам и целостность данных.
- **`FLASK_ENV=production`** включает `SESSION_COOKIE_SECURE` (cookie только по HTTPS).
- Healthcheck `/healthz`, авто-рестарт при сбое.

## Частые вопросы

**База пропала после редеплоя.** Не подключён Volume или не задан `DATABASE_URL`
на путь тома (вариант A), либо не подключён Postgres (вариант B). См. Шаг 3.

**Не могу войти после первого деплоя.** Посмотрите Deploy Logs — там
сгенерированный пароль администратора (если не задавали `ADMIN_PASSWORD`).

**Камера не открывается.** Убедитесь, что открываете именно `https://`-домен
Railway (не http). На Railway HTTPS включён по умолчанию.

**Нужно несколько инстансов / большая нагрузка.** Перейдите на PostgreSQL
(вариант B) — SQLite рассчитан на один инстанс. Также вынесите login-throttle в
Redis (см. `app/auth/routes.py`).

## Альтернатива: запуск через Docker

Railway не требует Dockerfile (используется Nixpacks). Если всё же нужен Docker,
минимальный образ:

```dockerfile
FROM python:3.12-slim
WORKDIR /app
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt
COPY . .
ENV FLASK_ENV=production
CMD python -m scripts.release && gunicorn 'app:create_app()' --bind 0.0.0.0:$PORT --workers 1 --threads 8 --timeout 60
```
